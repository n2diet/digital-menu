🔧 دليل حل مشاكل تشغيل الموقع

خطوات شاملة لتشخيص وإصلاح مشاكل عدم تشغيل المواقع

1
🚀 الفحص الأولي

⚠️ أولاً: تحقق من هذه النقاط

  • هل الرابط صحيح؟
  • هل الإنترنت يعمل؟
  • هل جربت متصفح آخر؟
  • هل جربت وضع Incognito؟
  • هل مسحت Cache المتصفح؟

🔍 خطوات الفحص السريع:

# فحص الاتصال ping your-domain.com # فحص DNS nslookup your-domain.com # فحص من متصفح آخر - جرب Chrome, Firefox, Safari - استخدم وضع Incognito/Private
2
🧹 مسح Cache والبيانات

مسح Cache المتصفح:

  • Chrome: Ctrl+Shift+Delete
  • Firefox: Ctrl+Shift+Delete
  • Safari: Cmd+Option+E

مسح DNS Cache:

# Windows ipconfig /flushdns # Mac sudo dscacheutil -flushcache # Linux sudo systemctl restart systemd-resolved
3
🌐 تغيير DNS

جرب تغيير DNS إلى خوادم عامة:

# Google DNS 8.8.8.8 8.8.4.4 # Cloudflare DNS 1.1.1.1 1.0.0.1 # OpenDNS 208.67.222.222 208.67.220.220

📦 فحص حالة النشر

❌ أخطاء النشر الشائعة:

  • Build failed - فشل في البناء
  • Deployment timeout - انتهاء وقت النشر
  • Domain not configured - الدومين غير مُعد
  • SSL certificate issues - مشاكل شهادة SSL

🔍 كيفية الفحص:

  • ادخل إلى لوحة تحكم Vercel/Netlify
  • تحقق من حالة آخر deployment
  • اقرأ رسائل الخطأ في Logs
  • تأكد من إعدادات الدومين
2
🔄 إعادة النشر

خطوات إعادة النشر:

# إذا كنت تستخدم Git git add . git commit -m "Fix deployment issues" git push origin main # أو من لوحة التحكم - اذهب إلى Deployments - انقر على "Redeploy" - انتظر اكتمال العملية

✅ علامات النجاح:

  • Build completed successfully
  • Deployment ready
  • Domain accessible
  • SSL certificate active
3
⚙️ إعدادات الدومين

تحقق من إعدادات الدومين:

  • Domain correctly added في لوحة التحكم
  • DNS records pointing to correct servers
  • SSL certificate generated
  • HTTPS redirect enabled
# فحص DNS records dig A your-domain.com dig CNAME www.your-domain.com # يجب أن تشير إلى: # Vercel: 76.76.19.61 أو cname.vercel-dns.com # Netlify: 75.2.60.5 أو netlify.app
1
🌍 فحص انتشار DNS

DNS قد يحتاج وقت للانتشار عالمياً (24-48 ساعة)

# فحص من مواقع مختلفة nslookup your-domain.com 8.8.8.8 nslookup your-domain.com 1.1.1.1 # أو استخدم أدوات online
فحص انتشار DNS DNS Checker
2
🔧 إصلاح DNS

⚠️ خطوات الإصلاح:

  1. تحقق من إعدادات DNS في مزود الدومين
  2. تأكد من صحة A records و CNAME
  3. احذف أي records قديمة أو متضاربة
  4. انتظر وقت الانتشار

إعدادات DNS الصحيحة:

# للدومين الرئيسي Type: A Name: @ Value: [IP address من مزود الاستضافة] # للـ www Type: CNAME Name: www Value: your-domain.com
1
🐛 أخطاء الكود الشائعة

❌ أخطاء شائعة:

  • Syntax errors في JavaScript/CSS
  • Missing dependencies في package.json
  • Wrong file paths
  • CORS issues
  • Environment variables missing

🔍 كيفية التشخيص:

  • افتح Developer Tools (F12)
  • تحقق من Console للأخطاء
  • راجع Network tab للطلبات الفاشلة
  • تحقق من Sources للملفات المفقودة
2
🔧 إصلاح أخطاء الكود

خطوات الإصلاح:

# فحص الكود محلياً npm install npm run dev # أو yarn install yarn dev # تحقق من الأخطاء في Terminal

✅ نصائح للوقاية:

  • اختبر الكود محلياً قبل النشر
  • استخدم linting tools
  • راجع dependencies بانتظام
  • احتفظ بنسخ احتياطية
🆘 مساعدة سريعة

إذا كان موقعك لا يعمل الآن:

  1. جرب رابط مختلف: استخدم الرابط المباشر من Vercel/Netlify
  2. انتظر قليلاً: قد يكون هناك تأخير في النشر
  3. تحقق من الحالة: زر صفحة status للمنصة
  4. اتصل بالدعم: إذا استمرت المشكلة
